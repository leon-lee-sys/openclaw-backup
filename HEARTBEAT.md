# HEARTBEAT.md

## 新闻早报质量规则（2026-04-12更新）

### 三先生要求
1. **不重复**：同一个新闻不要在同一次新闻报送中出现
2. **时效性**：所有内容必须是24小时之内的热点新闻
3. **有实质**：新闻要有时间、地点、人物，有热度和新颖度
   - 避免空洞无实质内容的新闻（如单纯"赛事持续进行"）
   - 每条新闻需包含具体人物、事件、数据

### 示例对比
❌ 不好："欧洲足坛多条战报持续更新"
✅ 好："皇马3-1击败巴萨，本泽马梅开二度"

❌ 不好："国际网球赛事持续进行，选手争夺排名"
✅ 好："德约科维奇晋级迈阿密公开赛决赛，将战阿尔卡拉斯"

## cron任务健康检查
每2小时检查一次，发现失败立即补救并报告；单次无问题不报告，每天晚10点集中报告一次。

### 检查命令
```bash
openclaw cron list 2>&1 | grep -E "error|fail"
```

### 失败处理
1. 记录失败任务到 memory/cron-failures.md
2. 尝试用 `openclaw cron run <job-id>` 重新触发
3. 如果重新触发成功，更新状态
4. 如果持续失败，发紧急通知给用户

### 报告规则
- ❌ 发现问题 → 直接处理，不报告
- ✅ 单次无问题 → 不报告
- 📅 每天晚10点 → 集中报告一次（无论有无问题）

### 记录文件
- 失败记录：memory/cron-failures.md
- 每日报告：memory/YYYY-MM-DD.md
